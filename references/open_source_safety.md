# public release safety

## 1. Core principle

This public package should share the method, not become a bundle of third-party examples.

Keep public:

- the skill logic
- the output template
- placeholder prompts
- your own banner and brand copy
- general structure

Do not keep public:

- third-party website names as official reusable examples
- third-party screenshots
- third-party recordings
- third-party copy fragments
- private paths
- private workflow notes that only make sense in one local setup

## 2. Example policy

1. Public example folders should default to blank templates.
2. If a sample prompt is needed, use placeholders.
3. Do not hardcode real site names into long-lived reference files.

## 3. Before publishing

Check at least:

1. third-party brand names
2. screenshots or recordings
3. absolute local paths
4. internal project codenames
5. private implementation notes that should stay local
